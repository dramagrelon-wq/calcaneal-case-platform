平台怎么用

  1. 医生完成实名注册和资质审核后进入病例系统。
  2. 新建病例，选择私有、管理员可见、圈内教学、公开教学或多中心研究。
  3. 上传术前、术中、术后和随访影像，做四点矫正、黑白增强和角度测量。
  4. 记录 Essex-Lopresti、Sanders、Zwipp、骨折脱位型和分型补充。
  5. 管理员或讲师在平台内跟帖讨论，沉淀教学意见和随访结果。

项目怎么一起维护

  1. 项目负责人每周在 GitHub Issues 发一个小主题。
  2. 讲师和维护者在 issue 下面评论临床建议、字段建议和隐私风险。
  3. 只提交脱敏示例、流程建议和界面反馈，不把真实患者资料放进公开仓库。
  4. 把有效评论整理成下一版任务，让 Codex 帮忙更新原型、文档和测试。
  5. 每 1-2 周做一次小发布，形成持续维护记录。

平台使用范围与医疗服务边界

本平台定位为医生之间的跟骨骨折病例采集、影像整理、教学讨论、随访管理和科研协作工具。平台内容不面向患者提供诊疗服务，不构成互联网诊疗、远程会诊、处方、医嘱或最终手术决策。用户不得以平台内容替代线下诊疗、病历记录、医院制度、医疗质量安全制度或本人依法执业责任。

患者隐私、病例资料与敏感信息保护

病例、影像、随访资料、身份证号、电话、病历号、住院号、面部图像、二维码、检查号、精确日期、罕见身份线索等均可能构成个人信息或敏感个人信息。共享病例、公开区、圈内讨论、普通导出和多中心研究数据不得包含可识别患者的信息。医生因随访需要记录的姓名、电话、身份证号等，应仅保存在“医生本地私密随访信息”栏，由医生本人在当前设备和浏览器内使用，不进入公开病例、圈内讨论、普通导出或科研数据集。正式联网部署前，应进一步落实授权、加密、访问控制、审计和删除机制。

多中心研究与伦理审批

申请进入多中心研究病例范围的用户，应提交伦理审批、机构资质、研究方案要求的授权或同意文件，并遵守数据最小必要、限定用途、访问控制、审计留痕和撤回/删除机制。未完成伦理和资质审核的病例不得进入科研采纳范围。

违规处理、留痕与追责

病例录入与隐私控制

私有

合并损伤

没有合并损伤可以留空；如有，请勾选部位并补充具体内容。

病例可见范围说明

实际可见范围请在左侧病例卡片中选择；这里用于说明各范围的含义。

私有

仅上传医生本人可见，适合个人随访、术后复盘和暂不分享的病例。

管理员可见

上传医生和认证管理员可见，适合希望获得私下教学指导但不想公开的病例。

圈内教学

在受限医生圈内讨论和教学，适合已经脱敏、希望获得更多同道意见的病例。

公开教学

对通过准入的医生公开展示，适合质量较高、已充分去标识化的教学病例。

医生本地私密随访信息

这些信息属于医生本人随访记录，只保存在当前设备和浏览器的本地存储；不进入普通病例导出，也不用于公开、圈内讨论或多中心研究展示。清理浏览器或更换设备可能导致丢失，正式部署时应增加加密、授权和删除机制。

批量归类

上传的普通图片会经 Canvas 重新编码，便于去除常见 EXIF metadata。DICOM 支持将在后续加入。

矫正

四点矫正：拖动四个角点贴住拍摄屏幕、片子或纸张的四个边角，再应用矫正；角点会限制在当前图片实际显示边界内。

采集清理

开启后在中间影像上拖出矩形区域，松手即保存为一个处理步骤；可用后退一步恢复。

滤镜调整

开启后在影像上左右拖动调对比度、上下拖动调亮度；滑块仍可精细调整，适合电脑端和后续移动端并存。

# 6 周维护与申请节奏

目标：用 6 周时间把项目从本地原型推进到一个有真实维护痕迹、可以申请 Codex for Open Source 的开源项目。

## 第 0 周：建仓库和邀请讲师

- 创建公开 GitHub 仓库
- 推送当前项目
- 邀请 2-4 位讲师
- 发讲师协作指引
- 创建第一批 issues

## 第 1 周：病例录入和隐私

主题：

- 病例字段是否够用
- 医生本地私密信息是否清楚
- 病例可见范围是否合理
- 隐私和脱敏提示是否足够

讲师任务：

- 每人评论 1 个字段或隐私相关 issue

## 第 2 周：影像处理和测量

主题：

- 影像分类
- 四点矫正
- 黑白增强
- Böhler 角、Gissane 角、轴位角度

讲师任务：

- 每人提出 1 条影像或测量体验建议

## 第 3 周：分型和随访

主题：

- Essex-Lopresti
- Sanders
- Zwipp
- 骨折脱位型
- 随访时间、终极指标、融合

讲师任务：

- 每人评论 1 个分型或随访相关 issue

## 第 4 周：管理员教学和病例质量

主题：

- 管理员可见
- 管理员申请
- 管理员如何私下指导病例
- 什么样的病例适合进入公开教学候选

讲师任务：

- 每人提出 1 条教学或病例质量建议

维护任务：

- 整理 v0.1.0 release notes
- 检查 README、Roadmap、Governance、Security、Contributing 是否完整

## 第 5 周：公开教学和多中心研究

主题：

- 精选教学病例
- 多中心研究候选标准
- 三步复位法教学表达

讲师任务：

- 每人推荐 1 个适合公开教学或研究候选的脱敏病例类型

## 第 6 周：申请准备

维护任务：

- 创建 `v0.1.0` release
- 整理维护者名单
- 截图或记录 issues、comments、commits、release
- 根据 `docs/CODEX_OSS_APPLICATION.md` 准备申请文本
- 提交 Codex for Open Source 申请

## 每周会议模板

时间：20-30 分钟

1. 本周完成了什么？
2. 讲师们提了哪些有效建议？
3. 哪些建议进入下一版？
4. 哪些涉及隐私、伦理或法律风险？
5. 下周每个人只做哪一件小事？

各位老师好，我正在做一个“跟骨骨折病例平台”的开源项目，目标是把病例采集、影像处理、角度测量、分型记录、教学讨论和随访管理放到一个更适合骨科医生使用的平台里。

这不是患者端，也不是让大家上传真实隐私病例。首轮只请大家做内部试用和提建议。

https://dramagrelon-wq.github.io/calcaneal-case-platform/

GitHub 仓库：
https://github.com/dramagrelon-wq/calcaneal-case-platform

## 完整版

各位老师好，我想邀请大家参与一个跟骨骨折病例平台的首轮内部试用。

这个项目的目标是：让骨科医生可以更方便地整理跟骨骨折病例、上传和处理影像、记录测量和分型、做病例讨论、完成随访记录。后续如果成熟，也可以逐步沉淀成教学病例库和研究数据基础。

现阶段大家不需要写代码，也不需要安装开发环境。最重要的是从医生使用习惯出发，指出它哪里不顺手、哪里缺字段、哪里容易误传隐私、哪里不符合教学讨论的真实场景。

体验地址：
https://dramagrelon-wq.github.io/calcaneal-case-platform/

GitHub 仓库：
https://github.com/dramagrelon-wq/calcaneal-case-platform

请大家本周每人至少完成一项：

1. 在 GitHub Issues 新建 1 条建议。
2. 在已有 issue 下评论 1 条临床意见。
3. 提供 1 个完全脱敏的教学病例思路。
4. 指出 1 个隐私、伦理或流程风险。
5. 试用后告诉我一个最影响使用体验的问题。

可以反馈的方向包括：

- 病例录入字段是否合理。
- 软组织损伤、合并伤、伴发疾病是否够用。
- 影像上传、批量归类、矫正、滤镜、测量是否顺手。
- Böhler 角、Gissane 角、轴位内外翻等记录方式是否符合习惯。
- Essex-Lopresti、Sanders、Zwipp 分型是否清楚。
- 随访节点、评分、终极指标是否合理。
- 病例讨论区是否能承担教学和带教功能。
- 哪些地方可能造成患者隐私泄露。

隐私要求请务必注意：

- 不上传姓名、电话、身份证号。
- 不上传住院号、门诊号、检查号、二维码。
- 不上传带 PACS 信息条的原始图片。
- 不上传可识别人脸或屏幕反光人物。
- 不在 GitHub issue 或群聊里发送未脱敏影像。

如果不熟悉 GitHub，可以先把建议发给我，我来帮忙整理成 issue。后面希望大家逐步习惯在 GitHub 上评论，因为这本身就是项目的维护记录。

## 给不会用 GitHub 的老师

如果你第一次用 GitHub，可以这样做：

1. 注册 GitHub 账号。
2. 把用户名发给我。
3. 打开项目仓库。
4. 点击 `Issues`。
5. 选择一个模板。
6. 写下你的建议。

不知道写什么，可以复制这个格式：

```text
我从临床角度建议：

1. 需要增加/修改：
2. 原因：
3. 对医生填写负担的影响：
4. 是否涉及隐私或伦理风险：
5. 是否适合进入下一版：
```

# 讲师协作指引

这份文档可以直接发给 2-4 位内部讲师。目标是让大家用最简单的方式参与项目维护，不要求会写代码。

## 你需要做什么

每周只需要完成一件小事，5-15 分钟即可：

- 评论一个 issue
- 提一个病例字段建议
- 补充一个分型说明
- 提醒一个隐私风险
- 建议一个随访指标
- 上传一个完全脱敏的教学病例思路
- 指出界面哪里不符合医生习惯

对不写代码的讲师来说，最核心的维护动作就是：在 GitHub 仓库的 `Issues` 里发一个主题，或者在别人发的主题下面评论。评论可以是临床建议、分型经验、隐私提醒、随访指标，也可以只是指出哪里不好用。

## 第一次参与

1. 注册 GitHub 账号。
2. 把 GitHub 用户名发给项目发起人。
3. 接受 GitHub 仓库邀请。
4. 进入项目仓库。
5. 点击 `Issues`。
6. 找一个你熟悉的话题，写一条评论。

评论可以很短，例如：

```text
我建议跟骨骨折病例录入里增加“皮肤软组织条件”，因为这会影响手术时机和入路选择。
```

或者：

```text
Zwipp 分型这里建议增加图示说明，不然年轻医生不容易填。
```

## 最简单的 4 类贡献

### 1. 病例字段建议

适合熟悉临床流程的老师。

可以回答：

- 这个字段临床上有没有必要？
- 有没有遗漏的重要变量？
- 这个字段医生愿不愿意填？
- 是否会增加太多负担？

### 2. 分型和测量建议

适合熟悉跟骨骨折分型和影像测量的老师。

可以回答：

- Essex-Lopresti 是否需要图示？
- Sanders 填写时最容易出错在哪里？
- Zwipp 哪些项目必须医生确认？
- Böhler 角、Gissane 角的测量提示是否够清楚？

### 3. 隐私和伦理提醒

适合谨慎、熟悉医院流程的老师。

可以回答：

- 哪些影像最容易暴露患者信息？
- 哪些内容不能进公开病例？
- 多中心研究需要哪些审批？
- 医生本地私密信息如何避免误传？

### 4. 教学病例建议

适合有病例积累的老师。

注意：不要在 GitHub issue 里上传真实患者可识别资料。

可以只写：

- 病例类型
- 教学价值
- 需要展示的影像阶段
- 复位关键点
- 随访结局

## 不需要做什么

- 不需要写代码
- 不需要安装开发环境
- 不需要处理 Git 命令
- 不要上传真实患者姓名、电话、身份证号、住院号、病历号
- 不要上传未脱敏影像

## 每周最低任务

每位讲师每周任选一项：

1. 评论 1 个 issue
2. 新建 1 个建议 issue
3. 审阅 1 个文档页面
4. 提供 1 个脱敏教学病例思路

## 推荐话术

如果你不知道怎么开始，可以复制下面的格式：

```text
我从临床角度建议：

1. 需要增加/修改：
2. 原因：
3. 对医生填写负担的影响：
4. 是否适合公开教学：
```

## 为什么需要你参与

这个项目不是单人软件，而是一个长期的骨科教学和病例协作平台。讲师的真实临床经验，会决定它是不是好用、可信、能长期维护。
